====================================================
  NATION CORE - UPDATE LOG v2.0.0-SNAPSHOT#35
====================================================

- NationCore.java & RepublicExecutiveOrdersMenu.java:
  → Updated comment references to nations/republic.yaml.

[CONFIG-DRIVEN MINISTER OFFICE CONSOLES]

[NEW FEATURE - PRESIDENTIAL REGISTRATION MENU (REPUBLIC ONLY)]
- Added a brand-new candidacy registration menu ("Presidential Registration") reachable during the
  REGISTRATION phase of a Republic election. It gathers six candidacy documents (berkas) from an
  aspiring candidate and files their candidacy when every document is in order.
- New file: gui/republic/RepublicElectionRegistrationGUI.java (follows the "Republic" naming prefix).
  → 54-slot menu. Title: "§8§lPresidential Registration".
  → Filler (LIGHT_BLUE_STAINED_GLASS_PANE) on the specified border slots
    0,1,2,3,5,6,7,8,9,17,18,26,27,35,36,44,45,46,47,48,50,51,52,53. Every remaining empty interior
    slot is ALSO filled so no items can ever be inserted (security).
  → Slot 4  — Information (GLOW_ITEM_FRAME): live registration-phase info (phase, time left,
    candidates X/2, how-to).
  → Slot 21 — Document · Playtime (BOOK → ENCHANTED_BOOK when met): realtime playtime requirement.
  → Slot 22 — Document · Experience Level (BOOK → ENCHANTED_BOOK when met): realtime level requirement.
  → Slot 23 — Document · Registration Fee (BOOK → ENCHANTED_BOOK when met): realtime Vault balance vs. fee.
  → Slot 30 — Document · Clean Record (BOOK → ENCHANTED_BOOK when met): realtime "no recent punishment" check.
  → Slot 31 — Document · Campaign Message (BOOK → ENCHANTED_BOOK when met): click to enter chat-input mode
    and write a campaign speech. The message is saved to PlayerData and reused as the candidate's campaign
    message in the following campaign phase; the button lore updates to preview the saved text. Character
    bounds are enforced (default 10-256 characters, configurable; the 256 max matches Minecraft's chat
    input cap so the whole message fits in one chat line).
  → Slot 32 — Document · Candidacy Agreement (BOOK → ENCHANTED_BOOK when accepted): a toggle (click to
    agree, click again to withdraw). Lore expands the three-point Presidential Candidate's Oath
    (win → fee to state treasury; lose → fee not refunded, still to state treasury; vow to uphold the law
    and serve honorably).
  → Slot 49 — Submit Candidacy (NETHER_STAR): only files the candidacy when all six documents are satisfied
    (all icons are ENCHANTED_BOOK / the agreement accepted); otherwise it reports the missing documents.
    Glows when ready.
  → Slot 43 — Back (SPECTRAL_ARROW): returns to the Republic Council main menu.

[MAIN MENU INTEGRATION]
- RepublicMainMenu.java:
  → Rewrote buildElectionCard(...) (President Election, slot 32) to be phase-aware. It now shows the live
    election status and updates its lore when the registration phase is open and running (Registration OPEN

[BACKEND - ELECTION MANAGER]
- ElectionManager.java:
  → Added submitCandidacy(Player, Nation, boolean agreementAccepted): the authoritative, Republic-only
    registration path. Re-validates every document server-side (playtime, level, clean record, campaign
    message word count, agreement, and finally the fee immediately before withdrawal), withdraws the
    registration fee, registers the candidate with their saved campaign message, and increments
    timesRanForPresident.
  → REQUIREMENT: once REGISTRATION_CANDIDATE_CAP (= 2) members have filed, registration auto-closes and the
    election advances straight to the CAMPAIGN phase.
  → Added realtime per-document helpers (meetsPlaytimeRequirement / meetsLevelRequirement /
    meetsFeeRequirement / meetsCleanRecordRequirement / hasValidCampaignMessage / meetsAllRegistrationDocuments)
    and config-driven character-bound getters (getCampaignMessageMinChars / getCampaignMessageMaxChars).
  → REMOVED the consecutive-term-limit / cooldown rule from the registration path (both the new flow and the
    legacy registerCandidate(Player, slogan)). The six documents are now the sole eligibility criteria.

[BACKEND - PERSISTENCE & CHAT INPUT]
- PlayerData.java:
  → Added persistent field presidentCampaignMessage (+ getter/setter/hasPresidentCampaignMessage) so a
    drafted campaign speech survives relogs and is reused in the next campaign phase.
- ChatListener.java:
  → Added pendingCampaignMessages input flow. Validates the character length, sanitizes the text (strips section
    signs and neutralizes '<'/'>' to prevent colour-code / MiniMessage injection into lore and broadcasts),

- GUIListener.java:
  → Registered the new menu (instance + click routing + drag-cancel) and added a transient, per-session

[SECURITY MECHANISMS]
- Every empty slot in the registration menu is filled and all clicks/drags are cancelled → items cannot be
  inserted or lost.
- All submission requirements are re-validated server-side in submitCandidacy (the GUI state is never trusted).
- Every registration click re-checks Republic type + nation membership + REGISTRATION phase, so a menu left
  open after the phase changes cannot be exploited.
- Double-registration, full-candidate-slot, and campaign-message word bounds are all enforced authoritatively.
- Campaign-message text is sanitized against formatting/tag injection.

[REPUBLIC-ONLY REFACTOR]
- The registration/candidacy flow (menu open, click handling, and submitCandidacy) is strictly gated to
  GovernmentType.REPUBLIC, and the President Election button rejects non-Republic contexts. This keeps the
  presidential election phase exclusive to Republic nations.

[FEE MECHANISM - NON-REFUNDABLE, FORFEIT TO STATE TREASURY]
- Reworked the registration fee to match the candidacy agreement: the fee is now NON-REFUNDABLE and, when
  the election concludes, every candidate's fee is forfeit into the nation's state treasury (win, lose, or
  inconclusive). The OLD deposit-refund mechanism (winner/loser refund rates paid back to the candidate's
  Vault) has been removed.
  → ElectionManager.concludeVoting(Nation): removed the refund loop; added collectRegistrationFees(...) which
    deposits each candidate's fee into nation.getTreasury() before the winner is resolved, and notifies the
    nation of the total collected.
  → ElectionManager.concludeVoting() (legacy global): refund loop replaced with a deposit into the global
    treasury.
  → Treasury.TransactionType: added REGISTRATION_FEE ("Candidate Registration Fee") for these treasury entries.
  → nations/republic.yaml: removed election.registration-refund-winner / registration-refund-loser.

[CONFIG]
- nations/republic.yaml:
  → Replaced the campaign-message word bounds with CHARACTER bounds:
    election.campaign-message.min-chars (10) and election.campaign-message.max-chars (256). The 256 default
    matches Minecraft's chat input cap so a candidate's whole campaign message fits in a single chat line.
