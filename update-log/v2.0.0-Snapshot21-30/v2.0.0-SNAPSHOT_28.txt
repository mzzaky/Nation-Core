====================================================
  NATION CORE - UPDATE LOG v2.0.0-SNAPSHOT#28
====================================================
Introduced a centralized factory-tax integration that lets FactoryCore forward the tax paid on any factory standing inside a nation's claimed territory straight into that nation's treasury. The feature is delivered as a one-directional soft dependency: NationCore exposes a small, stable, reflection-friendly public API and owns every security control, while FactoryCore binds to it without any shared build artifact. NationCore continues to build and run with zero knowledge of FactoryCore.

[NEW PUBLIC INTEGRATION API]
- NationCoreAPI.java (id.nationcore.api) — NEW:
  → Added a final, static-only facade that is the single supported entry point for other plugins.
  → Built exclusively from JDK/Bukkit types (Location, double, UUID, String) so consumers can bind reflectively, giving a true soft-depend with no shared jar and no load-order coupling beyond a softdepend entry.
  → Exposed `isFactoryTaxIntegrationEnabled()`, `getNationNameAt(Location)`, and `depositFactoryTax(Location, double, UUID, String)`.
  → Every method is null-safe and swallows all Throwables at the boundary, so a consumer can never be destabilised by NationCore's internal state (e.g. mid-disable or pre-init).

[CENTRALIZED FACTORY TAX HANDLER]
- FactoryTaxIntegration.java (id.nationcore.integration) — NEW:
  → Implements the actual routing: resolves the nation owning the land via TerritoryManager.getNationAt(Location) and deposits into its treasury through TreasuryManager.deposit(...) using TransactionType.TAX_INCOME.
  → Owns ALL integration policy on the treasury-owning side: master enable flag, revenue share percentage, and per-payment cap.
  → Tags every routed deposit in the treasury ledger with a "Factory tax (<factoryId>)" description and records the paying player for a full audit trail.

[SECURITY & ANTI-EXPLOIT CONTROLS]
- FactoryTaxIntegration.java (id.nationcore.integration):
  → Gate 1 — honours the master `integration.factory-tax.enabled` switch.
  → Gate 2 — rejects NaN, infinite, zero, and negative amounts before any money moves.
  → Gate 3 — rejects null locations and locations whose world is not loaded.
  → Gate 4 — routes nothing when the land is unclaimed (no owning nation).
  → Clamps the configurable revenue share to the 0-100 range and applies an optional hard per-payment cap to defend against treasury inflation from misconfigured/exploited factory prices.
  → Normalises floating-point dust (floored to 2 decimals) and re-validates the result is positive before depositing.
  → Wraps the treasury deposit in its own try/catch and logs failures without throwing, guaranteeing a consumer-side tax payment is never broken by a server-side error.

[CORE WIRING]
- NationCore.java (id.nationcore):
  → Added a `FactoryTaxIntegration` field, instantiated during onEnable() after the territory and treasury managers are ready.
  → Added the `getFactoryTaxIntegration()` getter consumed by NationCoreAPI.
  → The handler is created unconditionally and stays dormant until FactoryCore calls in and the operator enables it, so there is no startup cost or hard dependency.

[CONFIGURATION]
- config.yml (src/main/resources):
  → Added a new top-level `integration.factory-tax` block: `enabled`, `share-percent`, `max-per-payment`, and `log`.
  → Documented the deliberate "two-key" safety design — both NationCore and FactoryCore must enable the integration before any funds move.

[COMPATIBILITY]
- No existing behaviour, command, GUI, or save format was changed. With the partner plugin absent or the switch disabled, NationCore behaves identically to SNAPSHOT#27.
