====================================================
  NATION CORE - UPDATE LOG v2.0.0-SNAPSHOT#30
====================================================
Redesigned and updated the Republic Government GUI and the Executive Orders Menu layout, slot positions, materials, and confirmation security mechanics. The legacy multi-tab cabinet decision navigation within the Executive Orders Menu has been completely removed to dedicate the menu exclusively for presidential executive orders, styled with a modern, clean, and premium English lore design.

[REPUBLIC GOVERNMENT GUI SLOT RESTRUCTURE]
- RepublicGovernmentGUI.java (id.nationcore.gui.republic) — UPDATED:
  → Restructured the slot layouts to optimize navigation and layout balance:
    • Border Management: Moved from slot 21 to slot 30.
    • Member Management: Moved from slot 22 to slot 31.
    • Diplomacy Management: Moved from slot 23 to slot 32.
    • Executive Order: Moved from slot 30 to slot 39.
    • Start Arena (Presidential Games): Moved from slot 31 to slot 40.
    • Broadcast Message: Moved from slot 32 to slot 41.
    • Salary Claim: Moved from slot 48 to slot 34, and changed the material to EMERALD.
    • Set Announcement Message: Moved from slot 49 to slot 28, and changed the material to WRITABLE_BOOK.
    • Nation Settings: Moved from slot 50 to slot 37, and changed the material to COMMAND_BLOCK_MINECART.
  → Removed the legacy static Guide & Help button (originally at slot 37).
  → Added slots 48, 49, and 50 to the light blue stained glass pane filler array, replacing them with background pane fillers.
- RepublicGUIHandler.java (id.nationcore.gui.republic) — UPDATED:
  → Adjusted the slot click routing inside `handleGovernmentGUI` to align with the new slot indices and material checks (e.g. EMERALD on slot 34, COMMAND_BLOCK_MINECART on slot 37).

[EXECUTIVE ORDERS MENU REDESIGN & DEDICATION]
- RepublicExecutiveOrdersMenu.java (id.nationcore.gui.republic) — REWRITTEN:
  → Replaced the centralized multi-tab navigation style with a dedicated president-only Executive Orders layout. Removed all Cabinet Decision tabs (Defense, Treasury, Health, President) and associated decision-rendering logic.
  → Configured the new menu layout:
    • Filler: Placed light_blue_stained_glass_pane on slots 0-3, 5-9, 17-18, 26-27, 35-36, 44-48, and 50-53, creating a clean frame border.
    • Back: Placed spectral_arrow on slot 49 to return to the Republic Government GUI.
    • Information: Placed glow_item_frame on slot 4, displaying the nation name, tag, president name, and active decrees count.
    • Executive Orders: Placed on slots 20-24 and 29-33 (a 2x5 grid in the center).
  → Implemented dynamic item materials based on order status:
    • Material is ENCHANTED_BOOK (and glows) when the executive order is currently active.
    • Material is BOOK if the order is on cooldown, locked due to authorization, or insufficient funds.
    • Material is WRITABLE_BOOK when the order is ready and available to be issued.
  → Upgraded the description card lore with a premium English formatting structure utilizing Unicode dividers (`▬`), clear headers, and icons.

[DECREE CONFIRMATION SECURITY MECHANISM]
- RepublicGUIHandler.java (id.nationcore.gui.republic) — UPDATED:
  → Intercepted the click event for Executive Orders at slot 20-24 and 29-33 to check for `WRITABLE_BOOK` instead of `YELLOW_CONCRETE`.
  → Wrapped the decree issuance process inside a confirmation step using `gui.confirmActionGUI.open()`, preventing accidental activation and ensuring authorization verification before treasury withdrawal.
