[TAX & TREASURY GUI REDESIGN & CHAT DONATION SYSTEM]
- TaxGUI.java:
  → Redesigned the main menu using a 54-slot layout:
    - Filler (light_blue_stained_glass_pane) placed on slots 0-3, 5-9, 17, 18, 26, 27, 35, 36, and 44-53.
    - Balance Information (glow_item_frame) on slot 4, showing nation details, treasury balance, president, and members.
    - Total Income (warped_wart_block) on slot 21, displaying total nation income and the last 3 transaction records.
    - Donate button (writable_book) on slot 23 to trigger a chat-based donation process.
    - Total Expenses (nether_wart_block) on slot 30, displaying total nation expenses and the last 3 transaction records.
    - Transaction Logs (knowledge_book) on slot 32 to open the tax transaction history.
    - Back button (spectral_arrow) on slot 43 to return to the Republic Main Menu.
  → Added helper methods: buildBalanceInfoItem, buildTotalIncomeItem, buildTotalExpensesItem, buildDonateItem, and buildTransactionLogsItem.
  → Cleaned up legacy items (tax rate, debt status, pay debt button, debtors list, etc.).
- RepublicTreasuryMenu.java:
  → Redesigned both standard and legacy treasury menus using the same 54-slot layout and components (slots 4, 21, 23, 30, 32, and 43) as TaxGUI.
  → Redesigned the transaction logs layout to a 54-slot view, placing the Back button (spectral_arrow) on slot 49.
  → Improved fillTransactionSlots() to display transaction entries in chronological order (most recent first) with clean, modern lore styling.
- ChatListener.java:
  → Introduced `pendingTreasuryDonations` Map to track players currently entering a donation amount.
  → Implemented a chat-based donation flow inside onChat(): validates numeric inputs, withdraws funds via VaultHook, deposits them to the nation treasury under TransactionType.DONATION, updates/saves the nation database, and alerts members. Supports cancellation via "cancel" or "batal".
- CommonGUIHandler.java:
  → Updated handleTaxMenuGUI() to support the new 54-slot item interactions (Back, Donate chat trigger, and Logs).
  → Added a slot-aware overload for handleTreasuryGUI() to properly handle slots 43 (Back), 23 (Donate), and 32 (Logs).
  → Updated handleTreasuryTransactionsGUI() to handle navigation back via spectral_arrow instead of regular arrow.
- GUIListener.java:
  → Updated the calls to CommonGUIHandler.handleTreasuryGUI() to pass the clicked event slot.
